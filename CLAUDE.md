# AI PM 프로젝트 규칙

## 진행 원칙
- 단계적으로 프로젝트를 진행한다. 다음 단계로 넘어가기 전 반드시 현재 단계 완료를 확인한다.
- 작업 후 즉시 배포하지 않는다.
- 모든 기능 구현 후에는 TC(테스트 케이스)를 작성하고 코드 레벨 테스트를 수행한다.
- 테스트 수행 결과를 리포팅하며, 사용자가 직접 검증해야 할 항목을 명확히 안내한다.
- 산출물 및 서비스 기본 언어는 한글로 제공한다.
- 작업 완료 후에는 project_log.md 파일에 작업 내용을 기록하고 업데이트한다.

## 기술 스택
- Frontend: React, Vite
- DB/Auth: Supabase (MCP 연동)
- 배포: Vercel
- 소스코드: GitHub

## 환경변수 관리
- 모든 API Key, 비밀번호, 개인정보는 반드시 .env 파일로 관리한다.
- .env 파일은 절대 Git에 커밋하지 않는다. .gitignore에 반드시 포함한다.
- 배포 환경(Vercel)에서는 환경변수를 별도로 설정한다.

## 프로젝트 진행 단계
1. 프로젝트 초기화 및 규칙 정의
2. 에이전트 및 스킬 정의
3. DB 및 인증 연동 (Supabase)
4. 핵심 기능 구현 (채팅, 에이전트 호출, 평가)
5. 로컬 테스트 및 검증
6. GitHub 연동 및 Vercel 배포

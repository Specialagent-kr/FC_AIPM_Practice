# Jobs To Be Done 분석 (JTBD Framework)

## 스킬 설명
고객이 제품/서비스를 선택하는 이유를 기능적 욕구가 아닌 "해결하려는 일(Job)"의 관점에서 분석한다.
"사람들은 드릴을 사는 것이 아니라 벽에 구멍을 뚫는 것을 산다"는 원리를 기반으로 진짜 니즈를 발굴한다.

## 사용 시점
- 제품 기능이 아닌 고객 니즈 중심으로 사고 전환이 필요할 때
- 새로운 세그먼트나 사용 맥락을 발굴하고 싶을 때
- 기능 우선순위 결정의 근거가 필요할 때
- 경쟁사 대비 차별화 포인트를 재정의할 때

---

## 실행 지침

### Step 1: Job 인터뷰 또는 데이터 분석
고객이 제품을 사용하는 **전환 시점(Switching Moment)**에 집중:
- 처음 사용하기 시작한 계기는?
- 이전에는 어떻게 해결하고 있었나?
- 무엇이 기존 방법을 버리게 만들었나?
- 마지막으로 [제품]을 사용한 상황을 구체적으로 설명해달라

### Step 2: Job Statement 작성
**기본 구조:**
> "[상황]에서 [동기]를 가진 [고객]이 [목표]를 달성하고 싶다. 하지만 [장애물]이 있다."

**3가지 Job 유형:**
| 유형 | 설명 | 예시 |
|------|------|------|
| **Functional Job** | 실질적으로 해결하려는 과제 | "출장 경비 보고서를 빠르게 제출하고 싶다" |
| **Emotional Job** | 느끼고 싶은 감정 | "보고서를 실수 없이 제출했다는 안도감을 얻고 싶다" |
| **Social Job** | 타인에게 보이고 싶은 모습 | "꼼꼼한 팀원으로 인식되고 싶다" |

### Step 3: Job Map 작성
Job을 실행하는 단계별 프로세스 매핑:

```
[시작] → [준비] → [실행] → [모니터링] → [종료]
```

각 단계에서:
- 현재 어떻게 수행하는가?
- 어느 단계에서 가장 힘든가?
- 어느 단계에서 우리 제품이 개입하는가?

### Step 4: 니즈 스코어링
각 Job에 대해 **중요도 × 충족도** 기준으로 기회 점수 계산:

```
기회 점수 = 중요도 + max(중요도 - 충족도, 0)
```

- 중요도 높고 충족도 낮을수록 → 핵심 기회 영역
- 스코어 10점 이상: 즉각 집중

### Step 5: Job-to-Feature 매핑
도출된 핵심 Job을 현재/향후 기능과 연결:

| Core Job | 현재 기능 | 충족도 | 추가 필요 기능 |
|----------|---------|--------|-------------|
| [Job1] | [기능] | 중 | [제안] |
| [Job2] | 없음 | 저 | [제안] |

## 출력 형식 예시

```
## JTBD 분석 결과

### 핵심 Job Statement
"[상황]에서, [고객]은 [목표 달성]을 원한다. 하지만 [장애물] 때문에 어렵다."

### Job 유형 분류
- Functional: [내용]
- Emotional: [내용]
- Social: [내용]

### Job Map
준비 단계: [현재 행동 / Pain Point]
실행 단계: [현재 행동 / Pain Point]
완료 단계: [현재 행동 / Pain Point]

### 기회 스코어 Top 3
1. [Job] — 중요도: 9, 충족도: 3 → 기회 점수: 15
2. [Job] — 중요도: 8, 충족도: 5 → 기회 점수: 11
3. [Job] — 중요도: 7, 충족도: 4 → 기회 점수: 10

### 시사점 및 제품 방향
[도출된 인사이트와 제품 방향 제안]
```

## 품질 기준
- [ ] Job Statement가 기능이 아닌 고객의 목표로 기술된다
- [ ] Functional/Emotional/Social 3가지 유형이 모두 분석된다
- [ ] Job Map이 단계별로 구체적으로 작성된다
- [ ] 기회 점수가 정량적으로 산출된다
- [ ] Job-to-Feature 매핑으로 제품 방향이 연결된다

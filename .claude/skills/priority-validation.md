# 우선순위 검증 (Priority Validation)

## 스킬 설명
여러 기능, 프로젝트, 이슈 중 어떤 것을 먼저 해야 하는지 객관적인 기준으로 평가하고
이해관계자와 합의할 수 있는 근거를 제공하는 스킬이다.

## 사용 시점
- 로드맵에 넣을 기능의 우선순위를 결정해야 할 때
- 리소스가 제한된 상황에서 선택과 집중이 필요할 때
- 다양한 이해관계자의 요구사항을 조율해야 할 때
- 기술 부채, 버그, 신기능 간 상대적 중요도를 비교할 때

---

## 실행 지침

### Step 1: 우선순위 항목 목록화
평가 대상 기능/과제를 목록으로 정리하고 각 항목의 간단한 설명을 추가한다.

### Step 2: 프레임워크 선택

**RICE 스코어링** (정량적 평가에 적합)
```
RICE 점수 = (Reach × Impact × Confidence) / Effort
```
| 요소 | 정의 | 측정 방법 |
|------|------|---------|
| Reach | 얼마나 많은 사용자에게 영향을 미치나 (월간) | 영향받는 사용자 수 |
| Impact | 사용자에게 얼마나 큰 영향인가 | 0.25 / 0.5 / 1 / 2 / 3 |
| Confidence | 추정치에 얼마나 자신 있나 | 100% / 80% / 50% |
| Effort | 팀 전체 작업량 (월 단위) | person-month |

**ICE 스코어링** (빠른 의사결정에 적합)
```
ICE 점수 = Impact × Confidence × Ease
```
| 요소 | 척도 (1-10) |
|------|------------|
| Impact | 비즈니스/사용자에 미치는 영향 |
| Confidence | 이 추정이 맞다는 확신 |
| Ease | 구현 용이성 (반비례: 어려울수록 낮음) |

**MoSCoW** (범위 결정에 적합)
| 분류 | 의미 |
|------|------|
| Must Have | 이번 릴리스에 반드시 있어야 함 |
| Should Have | 중요하지만 없어도 릴리스 가능 |
| Could Have | 있으면 좋지만 우선순위 낮음 |
| Won't Have | 이번에는 하지 않기로 결정 |

### Step 3: 점수 산출 및 랭킹
선택한 프레임워크로 각 항목 점수를 계산하고 순위를 매긴다.

### Step 4: 전략적 검토 (Sanity Check)
수치 결과를 다음 기준으로 검토:
1. **전략 정렬**: 높은 점수 항목이 현재 회사/팀 OKR과 연결되는가?
2. **의존성**: 먼저 해야 하는 선행 작업이 있는가?
3. **기회비용**: 1순위를 선택함으로써 포기하는 것은 무엇인가?
4. **타이밍**: 시장 기회나 시즌성이 있는가?

### Step 5: 결과 문서화
우선순위 결정 근거를 기록하여 이해관계자 커뮤니케이션에 활용한다.

## 출력 형식 예시

```
## 우선순위 검증 결과 (RICE 기준)

### 평가 항목 및 점수
| 순위 | 기능 | Reach | Impact | Confidence | Effort | RICE 점수 |
|------|------|-------|--------|-----------|--------|---------|
| 1 | [기능A] | 5,000 | 2 | 80% | 1 | 8,000 |
| 2 | [기능B] | 2,000 | 3 | 50% | 0.5 | 6,000 |
| 3 | [기능C] | 10,000 | 1 | 100% | 3 | 3,333 |

### 전략적 검토
- 1순위 [기능A]: OKR "재방문율 향상"과 직결 → 유지
- 2순위 [기능B]: 선행 인프라 작업 2주 필요 → 1순위와 병렬 가능
- 3순위 [기능C]: 현 OKR과 연결 약함 → 다음 분기로 이동

### 최종 추천 순서
1. [기능A] — 착수 즉시
2. [기능B] — A와 병렬 진행
3. [기능C] — 다음 분기 재평가
```

## 품질 기준
- [ ] 평가 기준(프레임워크)이 상황에 맞게 선택된다
- [ ] 점수 산출 기준이 팀 내 합의된다
- [ ] 전략 정렬 및 의존성 검토가 포함된다
- [ ] 최종 결정 근거가 문서화된다
- [ ] Won't Have / 제외 항목의 이유가 명시된다

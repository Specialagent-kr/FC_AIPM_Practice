# 사용자 인터뷰 설계 및 분석 (User Interview)

## 스킬 설명
고객의 실제 행동, 니즈, Pain Point를 깊이 이해하기 위한 정성 조사 기법이다.
가설 검증이 아닌 새로운 인사이트 발견에 집중하며, 편향 없는 질문 설계와 체계적인 분석 방법을 제공한다.

## 사용 시점
- 새로운 제품/기능 개발 전 사용자 니즈를 탐색할 때
- 기존 기능의 사용 패턴과 이유를 이해할 때
- 사용자 페르소나를 구체화할 때
- 정량 데이터의 원인을 정성적으로 파악할 때

---

## 실행 지침

### Step 1: 인터뷰 목적 정의
1. **핵심 학습 목표** 3가지 이내로 정의
   - "이 인터뷰를 통해 [무엇을] 알고 싶은가?"
2. **가설** 명시 (검증이 아닌 탐색 목적임을 인식)
3. **대상 사용자** 정의 (세그먼트, 경험 수준, 모집 기준)
4. **인터뷰 수** 결정 (일반적으로 세그먼트당 5-8명)

### Step 2: 스크리닝 질문 작성
인터뷰 전 적합한 대상자를 선별하는 3-5개 질문:
- 사용 경험 여부
- 빈도/강도
- 특정 행동 여부 (예: 특정 기능 사용자)

### Step 3: 인터뷰 가이드 작성

**구조 (60-90분 기준):**
| 구간 | 시간 | 내용 |
|------|------|------|
| 아이스브레이킹 | 5분 | 자기소개, 인터뷰 목적 설명 |
| 배경 파악 | 10분 | 직업, 관련 행동 패턴 |
| 핵심 탐색 | 40분 | 주요 질문 (행동 중심) |
| 깊이 파기 | 15분 | Follow-up 및 예외 탐색 |
| 마무리 | 5분 | 추가 의견, 다음 단계 안내 |

**좋은 질문 원칙:**
- ✅ 과거 행동 중심: "마지막으로 [행동]을 했을 때를 얘기해주세요"
- ✅ 개방형: "어떻게", "왜", "그때 어떤 생각을 하셨나요"
- ❌ 유도형 금지: "~가 불편하지 않으셨나요?"
- ❌ 미래 의향 금지: "이런 기능이 있으면 쓰실 건가요?"

### Step 4: 인터뷰 진행 가이드
- 5초 침묵 법칙: 답변 후 5초 기다렸다가 Follow-up
- "왜요?" 3번 법칙: 표면 답변 아래 진짜 이유 탐색
- 노트 테이커 분리 권장 (진행자 집중)
- 녹음/전사 동의 확보

### Step 5: 인사이트 분석 (인터뷰 후)
1. **전사** 또는 핵심 발언 기록
2. **어피니티 다이어그램**: 유사 발언 그룹핑
3. **패턴 도출**: 반복 언급되는 니즈/Pain Point
4. **인사이트 카드** 작성 (관찰 → 해석 → 시사점)
5. **페르소나 또는 Jobs To Be Done** 업데이트

## 출력 형식 예시

```
## 인터뷰 설계 결과

### 학습 목표
1. 현재 [행동]의 주요 Pain Point는 무엇인가?
2. [행동]을 할 때 어떤 대안을 사용하는가?
3. 이상적인 해결 방식은 어떤 모습인가?

### 스크리닝 질문
Q1. 최근 3개월 내 [제품/서비스]를 사용한 경험이 있으신가요?
Q2. 얼마나 자주 [행동]을 하시나요?

### 인터뷰 가이드 (핵심 질문)
1. [행동]을 마지막으로 했을 때 과정을 처음부터 끝까지 설명해주세요.
2. 그 과정에서 가장 힘들었던 부분은 어디였나요?
3. 그때 어떻게 해결하셨나요?
4. 이상적으로는 어떻게 되면 좋겠다고 생각하셨나요?
5. 지금 쓰고 있는 다른 방법이나 도구가 있나요?

### 분석 프레임워크
- 관찰: [사용자가 한 말/행동]
- 해석: [우리가 내린 해석]
- 시사점: [제품에 주는 시사점]
```

---

## 품질 기준
- [ ] 학습 목표가 3개 이내로 명확히 정의된다
- [ ] 질문이 과거 행동 중심으로 작성된다
- [ ] 유도형/미래 의향 질문이 없다
- [ ] 인터뷰 대상이 구체적으로 정의된다
- [ ] 분석 시 관찰-해석-시사점이 분리된다

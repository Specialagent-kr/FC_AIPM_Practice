# UT 계획 (Usability Test Planning)

## 스킬 설명
실제 사용자가 제품을 사용하는 과정을 관찰하여 UI/UX 문제점을 발견하고
개선 방향을 도출하는 사용성 테스트를 체계적으로 설계하는 스킬이다.

## 사용 시점
- 신규 기능 출시 전 사용성을 검증해야 할 때
- 디자인 시안에 대한 사용자 반응을 확인할 때
- 기존 기능의 이탈/실패율이 높은 원인을 파악할 때
- 온보딩 플로우나 핵심 전환 경로를 최적화할 때

---

## 실행 지침

### Step 1: 테스트 목적 및 범위 정의
1. **핵심 질문** 2-3개 설정
   - "사용자가 [기능]을 처음 사용할 때 어떤 어려움을 겪는가?"
2. **테스트 범위** 명확화
   - 테스트할 화면/플로우 특정
   - 제외 영역 명시
3. **테스트 방식** 선택

| 방식 | 특징 | 적합한 상황 |
|------|------|----------|
| 모더레이티드 | 진행자 동석, 실시간 질문 | 복잡한 플로우, 깊이 있는 이해 필요 |
| 언모더레이티드 | 사용자 단독 수행 | 빠른 피드백, 대규모 참여자 |
| 원격 | 화상으로 진행 | 지리적 제약, 비용 절감 |
| 대면 | 직접 관찰 | 신체 반응, 표정 관찰 중요 |

### Step 2: 참가자 모집 계획
1. **참가자 프로필** 정의
   - 타겟 사용자 특성 (경험 수준, 인구통계 등)
   - 스크리닝 기준
2. **참가자 수**: 세그먼트당 5-8명 (Jakob Nielsen의 5명 법칙 기준)
3. **모집 방법**: 사용자 인터뷰 풀, 소셜 모집, 리쿠르팅 서비스

### Step 3: 테스트 태스크 설계

**좋은 태스크 작성 원칙:**
- ✅ 현실적인 시나리오 기반 ("방금 친구에게 상품을 추천받았습니다...")
- ✅ 완료 가능한 명확한 목표 제시
- ✅ 방법이 아닌 목표만 기술 ("결제 버튼을 클릭하세요" ❌ → "이 상품을 구매해보세요" ✅)
- ❌ 단계 힌트 포함 금지
- ❌ 시스템 용어 사용 금지

**태스크 구성 (60-90분 기준):**
1. 워밍업 태스크 (1개): 부담 없는 탐색 과제
2. 핵심 태스크 (3-5개): 검증하려는 핵심 플로우
3. 심화 태스크 (1-2개): 엣지 케이스 또는 오류 복구

### Step 4: 측정 지표 정의

**정량 지표:**
| 지표 | 측정 방법 |
|------|---------|
| 태스크 성공률 | 완료/미완료 비율 |
| 소요 시간 | 태스크 시작~완료 시간 |
| 오류 횟수 | 잘못된 클릭/입력 횟수 |
| 도움 요청 횟수 | 힌트 요청 횟수 |

**정성 지표:**
- 사고 발화(Think Aloud) 중 언급된 어려움
- 감정 반응 (표정, 탄식, 긍정 발화)
- 마무리 인터뷰 피드백
- SUS (System Usability Scale) 점수

### Step 5: 분석 및 결과 보고 계획
1. **심각도 분류** (1-4등급):
   - P1: 치명적 — 태스크 완료 불가
   - P2: 심각 — 태스크 완료에 큰 어려움
   - P3: 보통 — 불편하지만 완료 가능
   - P4: 경미 — 개선 권장
2. **어피니티 다이어그램**으로 문제 그룹핑
3. 우선순위별 개선 권고사항 도출

## 출력 형식 예시

```
## UT 계획서: [기능/제품명]

### 테스트 목적
- 핵심 질문 1: [내용]
- 핵심 질문 2: [내용]

### 참가자
- 대상: [프로필]
- 인원: [N]명 (세그먼트 A: [n]명, 세그먼트 B: [n]명)
- 방식: [모더레이티드 / 원격 등]

### 태스크 목록
| # | 태스크 시나리오 | 성공 기준 | 예상 소요 시간 |
|---|-------------|---------|------------|
| T1 | [시나리오] | [기준] | 5분 |
| T2 | [시나리오] | [기준] | 10분 |
| T3 | [시나리오] | [기준] | 8분 |

### 측정 지표
- 정량: 태스크 성공률, 평균 소요 시간, SUS 점수
- 정성: Think Aloud 내용, 마무리 인터뷰

### 일정
- 모집: [날짜]
- 테스트: [날짜]
- 분석/보고: [날짜]
```

## 품질 기준
- [ ] 테스트 목적이 2-3개 핵심 질문으로 명확히 정의된다
- [ ] 태스크가 시나리오 기반으로 작성되며 힌트가 없다
- [ ] 정량/정성 지표가 모두 포함된다
- [ ] 심각도 분류 기준이 사전 정의된다
- [ ] 테스트 일정(모집-진행-분석)이 구체화된다

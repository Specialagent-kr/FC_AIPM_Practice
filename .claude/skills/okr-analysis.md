# OKR 분석 및 정렬 (OKR Analysis)

## 스킬 설명
조직/팀/개인의 목표를 OKR 프레임워크로 구조화하고, 상위 목표와의 정렬을 검증하며,
측정 가능한 핵심 결과(Key Results)를 도출하는 스킬이다.

## 사용 시점
- 분기/연간 OKR 수립이 필요할 때
- 기존 OKR의 품질을 검토하고 개선할 때
- 제품 목표와 비즈니스 목표의 정렬을 확인할 때
- 팀 OKR을 상위 회사 OKR에 연결할 때

---

## 실행 지침

### Step 1: 컨텍스트 파악
1. OKR을 수립하는 레벨 확인 (회사 / 팀 / 개인)
2. 기간 확인 (연간 / 분기)
3. 상위 목표(회사/팀 OKR)가 있으면 입력받는다
4. 현재 주요 과제나 기회를 3-5가지 파악한다

### Step 2: Objective 수립
좋은 Objective의 조건:
- **정성적이고 영감을 주는** 문장 (숫자 포함 금지)
- 달성 시 팀/조직에 어떤 변화를 가져오는지 명확
- 1개 분기에 집중할 수 있는 범위
- 동사로 시작 (예: "확보하다", "달성하다", "전환하다")

Objective 초안 3개를 제안하고 최종 선택을 돕는다.

### Step 3: Key Results 도출
각 Objective에 대해 KR 2-4개 작성:

좋은 Key Result의 조건:
- **정량적이고 측정 가능** (숫자/비율/날짜 포함)
- Objective 달성 여부를 직접 증명
- 현재 수치(Baseline)와 목표 수치 명시
- Lagging 지표 중심, 필요시 Leading 지표 보완

❌ 나쁜 예: "사용자 만족도를 높인다"
✅ 좋은 예: "NPS를 32점에서 50점으로 향상시킨다"

### Step 4: OKR 품질 검증 (CRAFT 체크)
- **C**lear: 명확하게 이해되는가?
- **R**elevant: 상위 목표와 연결되는가?
- **A**mbitious: 도전적이지만 달성 가능한가? (70% 달성 기준)
- **F**ocused: 1-3개 Objective, 각 2-4개 KR로 집중되어 있는가?
- **T**ime-bound: 기간이 명확한가?

### Step 5: 정렬 확인
- 하위 팀 OKR → 상위 팀 OKR → 회사 OKR 연결 맵 작성
- 누락된 연결고리 또는 상충 요소 식별

## 출력 형식 예시

```
## OKR (2026년 Q1)

### Objective 1
"고객이 자발적으로 추천하는 제품으로 성장한다"

**Key Results:**
| KR | 현재(Baseline) | 목표 | 측정 방법 |
|----|--------------|------|---------|
| KR1: NPS 점수 향상 | 32점 | 50점 | 분기 NPS 설문 |
| KR2: 재방문율 향상 | 40% | 60% | 앱 분석 툴 |
| KR3: 사용자 추천 가입 비율 | 15% | 30% | 가입 경로 추적 |

**CRAFT 검증:**
- Clear: ✅ 명확
- Relevant: ✅ 회사 OKR "사용자 기반 확대"와 연결
- Ambitious: ✅ 도전적이나 달성 가능 (70% 기준)
- Focused: ✅ 3개 KR로 집중
- Time-bound: ✅ Q1 (1-3월)

**리스크:** KR3는 추적 인프라 구축 필요 (2주 소요 예상)
```

---

## 품질 기준
- [ ] Objective는 정성적이고 숫자를 포함하지 않는다
- [ ] 각 KR은 Baseline 수치를 포함한다
- [ ] KR은 Activity(행동)가 아닌 Outcome(결과)을 측정한다
- [ ] CRAFT 기준 전항목을 검증한다
- [ ] 상위 OKR과의 연결이 명시된다

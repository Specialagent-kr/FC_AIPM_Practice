# 아이디어 진단 및 검증 (Idea Diagnosis)

## 스킬 설명
새로운 제품 아이디어나 기능 아이디어를 체계적으로 분석하여 타당성을 평가하고,
최소한의 비용으로 빠르게 검증할 수 있는 방법을 설계하는 스킬이다.

## 사용 시점
- 새로운 아이디어를 개발에 들어가기 전 사전 검증할 때
- 여러 아이디어 중 우선순위를 결정해야 할 때
- 가설의 리스크를 사전에 파악하고 싶을 때
- 최소한의 투자로 시장 반응을 확인하고 싶을 때

---

## 실행 지침

### Step 1: 아이디어 명확화
1. **아이디어를 1문장으로** 정의한다
   > "[타겟 사용자]가 [문제]를 해결하기 위해 [솔루션]을 사용한다"
2. 해결하는 **핵심 문제** 명시
3. 기존 대안 대비 **차별점** 서술

### Step 2: 가설 도출
아이디어에 내포된 전제(가정)를 3가지 유형으로 분류:

| 유형 | 질문 | 예시 |
|------|------|------|
| **고객 가설** | 이 문제를 가진 고객이 실제로 존재하는가? | "직장인은 점심 메뉴 선택에 피로감을 느낀다" |
| **문제 가설** | 이것이 진짜 해결해야 할 문제인가? | "메뉴 선택 시간이 10분 이상 걸린다" |
| **솔루션 가설** | 제안한 방식이 문제를 해결하는가? | "AI 추천으로 선택 시간을 2분으로 줄일 수 있다" |

### Step 3: 리스크 진단 (Riskiest Assumption)
1. 도출된 가설을 **검증 비용 × 실패 시 영향도** 기준으로 평가
2. **가장 위험한 가설(Riskiest Assumption) 1개** 선정
3. 이 가설이 틀렸을 때 아이디어가 무너지는지 확인

### Step 4: 검증 방법 설계
리스크 수준에 따라 적절한 검증 방법 선택:

| 방법 | 소요 시간 | 비용 | 검증 대상 |
|------|----------|------|---------|
| 고객 인터뷰 (5-8명) | 1-2주 | 저 | 고객/문제 가설 |
| 랜딩 페이지 + 사전 등록 | 1주 | 저 | 수요 가설 |
| Wizard of Oz (수동 운영) | 2-4주 | 중 | 솔루션 가설 |
| Concierge MVP | 4-8주 | 중 | 전체 가설 |
| 프로토타입 테스트 | 2-4주 | 중 | UX/솔루션 |
| 파일럿 출시 | 8-12주 | 고 | 비즈니스 모델 |

### Step 5: 검증 기준 설정 (Success Criteria)
검증 성공/실패를 판단하는 **정량적 기준** 미리 설정:
- "인터뷰 대상 5명 중 4명 이상이 이 문제를 '심각하다'고 표현"
- "랜딩 페이지 전환율 10% 이상"

## 출력 형식 예시

```
## 아이디어 진단 결과

### 아이디어 정의
[타겟] [문제] [솔루션] 한 문장

### 핵심 가설
| 유형 | 가설 | 위험도 |
|------|------|--------|
| 고객 가설 | [내용] | 상/중/하 |
| 문제 가설 | [내용] | 상/중/하 |
| 솔루션 가설 | [내용] | 상/중/하 |

### 최우선 검증 가설 (Riskiest Assumption)
[가설 내용]
→ 이 가설이 틀리면: [아이디어에 미치는 영향]

### 추천 검증 방법
방법: [인터뷰 / 랜딩페이지 / MVP 등]
기간: [N주]
성공 기준: [정량적 기준]

### 리스크 및 고려사항
- [리스크 1]
- [리스크 2]
```

---

## 품질 기준
- [ ] 아이디어가 타겟-문제-솔루션 구조로 명확히 정의된다
- [ ] 가설이 고객/문제/솔루션 3가지 유형으로 분류된다
- [ ] Riskiest Assumption이 명확히 선정된다
- [ ] 검증 방법은 리스크 수준에 적합하다
- [ ] 성공 기준이 정량적으로 사전 정의된다
